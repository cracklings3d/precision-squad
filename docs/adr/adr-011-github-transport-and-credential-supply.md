# ADR-011: GitHub Transport and Credential Supply## StatusAccepted## Date2026-06-13## ContextV1 of precision-squad requires clear definitions for how GitHub operations are performed and how credentials are supplied. The system must support multiple transport mechanisms (MCP and CLI) while maintaining a simple, PAT-only credential model. These decisions were needed to establish the execution contract for GitHub interactions and to keep the credential supply orthogonal to the transport mechanism.## Decision### Transport Model- `GITHUB_TRANSPORT=auto` — probe MCP first; fall back to `gh` CLI; error if neither available- `GITHUB_TRANSPORT=mcp` — require MCP; error if unavailable- `GITHUB_TRANSPORT=cli` — require `gh` CLI; error if unavailable### Credential Supply- PAT-only credential supply for GitHub operations- Token resolution: `GITHUB_TOKEN` (project) takes precedence over `OpenCode_Github_Token` (system-managed)### IndependenceThe transport model is independent of how credentials are supplied (PAT). MCP availability is probed once per run and cached.### V1 ScopeGitHub App authentication is explicitly out of scope for V1.### Execution Model- CLI/service first- filesystem-backed run persistence- docs-first execution wrapped behind a narrow adapter boundary## RationaleSeparating transport from credential supply allows the system to remain flexible with how it reaches GitHub while keeping the authentication model simple and predictable. The `auto` default provides good ergonomics by probing available tools, while explicit `mcp` and `cli` modes allow operators to pin to a specific transport when needed. PAT-only authentication reduces complexity and aligns with the docs-first philosophy of keeping things explicit and auditable. Probing MCP once per run and caching avoids repeated overhead. The filesystem-backed persistence and narrow adapter boundary ensure that execution remains traceable and independent of specific tool internals.## Consequences- Operators can choose between MCP and CLI transports without changing how credentials are configured- PAT tokens must be available via `GITHUB_TOKEN` or `OpenCode_Github_Token`- MCP availability is determined once at the start of a run and reused- GitHub App authentication will require a future ADR to extend the credential supply model- The narrow adapter boundary means transport logic is isolated from business logic## References- [README.md](../../README.md)- [instructions/github-transport.md](../../instructions/github-transport.md)- [Issue #186](https://github.com/cracklings3d/precision-squad/issues/186)
